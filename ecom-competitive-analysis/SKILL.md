# E-Commerce Competitive Analysis v3.0

给定一个电商商品链接，执行完整的竞品分析工作流，产出可直接支撑投放决策的结构化报告。

## 核心设计原则

1. **能拿到真数据的地方绝不靠推理** — 优先使用页面提取、SimilarWeb、Ad Library、Google Trends 等公开数据源
2. **必须推理的地方给出验证路径** — 每个推理结论都附带"怎么用最小成本验证"
3. **分层输出，按需消费** — 一页纸决策摘要 → 竞品速览 → 深度报告，用户可以在任何层级停下
4. **数据分层获取** — 自动获取层（免费公开数据）→ 用户输入层（引导提供订单/广告数据）→ 付费测试层（设计验证方案）

## 整体流程

```
用户输入商品链接（+可选：指定竞品、选择分析深度）
      ↓
CP0 → 询问分析深度 + 是否有历史数据可提供
      ↓
Phase 1: 目标商品数据提取 + Google Trends 品类热度 + SimilarWeb 受众基准
      ↓
CP1 → 确认商品信息和分析方向（含受众基准数据）
      ↓
Phase 2: 竞品发现（多源交叉：平台内搜 + 搜索引擎 + Ad Library）
      ↓
CP2 → 展示候选竞品清单，用户确认/增减
      ↓
Phase 3: 竞品数据采集（页面提取 + 评论采样 + 广告素材查询 + SimilarWeb竞品受众对比）
      ↓
Phase 4: 产出一 — 一页纸决策摘要
Phase 5: 产出二 — 竞品速览卡片 + 横向对比 + 素材对比图板
Phase 6: 产出三 — 深度四模块分析报告
      ↓
CP3 → 确认行动优先级，是否需要深挖某个模块
```

**CP = Checkpoint（检查点）**，在关键节点暂停确认方向，避免跑完整份报告才发现偏了。

## CP0：分析深度选择 + 数据摸底

在收到用户的商品链接后，先询问：

```
"你想要哪种分析深度？"

A）快速扫描（5-10分钟）
   → 商品信息 + 竞品速览卡片 + 横向对比表 + Top 3 行动建议
   → 适合：选品调研、快速摸底

B）标准分析（15-25分钟）【默认】
   → 完整三部分报告

C）深度分析（30-50分钟）
   → 标准 + Ad Library 广告素材分析 + 评论深度采样 + 素材对比图板
   → 适合：重点产品上架前的全面准备

---

"另外，你手上有以下任何数据吗？有的话可以让分析更精准：

① 历史订单数据（CSV/Excel）→ 我会提供模板告诉你需要哪些字段
② 广告后台数据（Facebook/Google Ads 截图或导出）
③ GA4 网站受众数据

没有也完全OK，我会用公开数据源做最佳分析。"
```

如果用户没有明确选择深度，默认使用 B 标准分析。

## Phase 1：目标商品数据提取 + 受众基准

### 1.1 页面信息提取

访问用户提供的商品链接，提取以下信息到结构化表格：

| 字段 | 采集内容 |
|------|---------|
| 商品名称 | 完整标题 |
| 品牌/店铺 | 店铺名、品牌标识 |
| URL | 原始链接 |
| SKU/ID | 如页面可见 |
| 价格 | 现价、原价、折扣、分期方案 |
| 品类路径 | 面包屑导航 |
| 材质/规格 | 材质、尺寸、重量 |
| 评分/评价 | 星级、评价数量 |
| 配送 | 时效、免运费门槛 |
| 标签 | "NEW"、"BESTSELLER"、促销活动 |
| 商品描述 | 核心卖点、文案语气、叙事角度 |
| 交叉推荐 | "经常一起买"、搭配推荐 |

**提取技巧：**
- 优先用 `get_page_text` 或 `javascript_tool` 提取结构化内容
- 页面过大时用 `javascript_tool` + `document.querySelector` 定向查询
- 原样记录商品描述文案 — 措辞对后续分析很重要
- 记录缺失项（如无评价 = 新品信号）

### 1.2 Google Trends 品类热度

访问 Google Trends 查询目标商品的核心品类关键词：

```
步骤：
→ 访问 https://trends.google.com/trends/explore?q=[核心品类词]&geo=US（或目标市场）
→ 提取或通过搜索获取：
  - 12个月搜索趋势方向（上升/平稳/下降）
  - 季节性波动模式（哪几个月是高峰）
  - 相关查询词（rising queries）
→ 如有多个关键词候选，叠加对比
```

输出格式：
```
品类热度判断：
- 趋势方向：[上升 📈 / 平稳 ➡️ / 下降 📉]
- 季节性：[描述高峰月份]
- 相关热门查询：[列出 3-5 个]
- 数据来源：Google Trends（可信度 ★★★★☆）
```

### 1.3 SimilarWeb 受众基准数据（v3 新增 — 必做）

> 这是 v3 最重要的新增步骤。SimilarWeb 提供免费的网站受众数据，可以直接获取真实的年龄/性别/流量分布，避免纯推理。

```
步骤：
→ 提取品牌域名（如商品在 zgallerie.com → 用 zgallerie.com）
→ 访问 https://www.similarweb.com/website/[品牌域名]/
→ 提取以下数据到结构化表格：

必提取（首页即可获取）：
  · 月访问量（Total Visits）
  · 跳出率（Bounce Rate）
  · 页面/访次（Pages per Visit）
  · 平均访问时长（Avg Visit Duration）
  · 全球排名 / 国家排名 / 品类排名

→ 访问 https://www.similarweb.com/website/[品牌域名]/#demographics
→ 提取受众数据：
  · 性别分布（Male/Female %）
  · 年龄分布（18-24 / 25-34 / 35-44 / 45-54 / 55-64 / 65+ 各占比%）

→ 访问 https://www.similarweb.com/website/[品牌域名]/#traffic-sources
→ 提取流量来源：
  · Top 5 流量来源渠道及占比
  · 有机搜索 vs 付费搜索占比
  · Top 搜索关键词（品牌词 vs 非品牌词）

→ 提取相似网站（Similar Sites）Top 5
→ 如有公司信息（Company Info），提取员工规模和收入区间
```

> **注意：** SimilarWeb 免费版有访问次数限制。如果页面无法加载或返回付费墙，改用以下降级方案：
> - 搜索 "site:similarweb.com [品牌域名]" 获取缓存数据
> - 搜索 "[品牌名] website traffic demographics" 获取第三方引用数据
> - 如果完全无法获取，在报告中明确标注"SimilarWeb数据未获取，以下为纯推理"

输出格式：
```
品牌受众基准（SimilarWeb）：
- 月访问量：[X]K
- 性别：女性 [X]% / 男性 [X]%
- 年龄分布：
  · 18-24: [X]% | 25-34: [X]% | 35-44: [X]%
  · 45-54: [X]% | 55-64: [X]% | 65+: [X]%
- 最大受众群：[XX-XX岁Y性，占比Z%]
- 流量来源 Top 3：[渠道 X%]、[渠道 X%]、[渠道 X%]
- 相似网站：[列出 Top 5]
- 数据来源：SimilarWeb（可信度 ★★★★☆）

⚠️ SimilarWeb vs 页面推断校验：
- [列出SimilarWeb数据与页面风格推断的一致/冲突点]
- 如有冲突，以SimilarWeb数据为准（样本量更大）
```

## Phase 2：竞品发现（多源交叉）

### 2.1 发现路径

同时使用以下路径，取并集后去重：

**路径 A — 用户指定竞品**（可信度 ★★★★★）
- 用户提供的竞品链接或名称，直接纳入

**路径 B — 平台内搜索**（可信度 ★★★★☆）
```
→ 访问目标商品所在平台（如 Amazon.com / AliExpress / Etsy）
→ 搜索目标商品的核心品类关键词
→ 提取搜索结果前 10-20 个商品：名称、价格、评分、评价数、是否 Sponsored
→ 筛选出最具竞争力的 3-5 个
```

**路径 C — 搜索引擎交叉验证**（可信度 ★★★☆☆）
```
→ 搜索 "[产品类型] + best / top / vs / alternative / review"
→ 查找评测文章、对比榜单、用户讨论
→ 覆盖 Amazon、Etsy、eBay、独立站、垂直零售商
```

**路径 D — Facebook Ad Library 广告竞品**（可信度 ★★★☆☆）
```
→ 访问 https://www.facebook.com/ads/library/
→ 选择 "All ads" 类别
→ 搜索品类关键词
→ 筛选正在投放广告的品牌/商家
→ 提取：广告主名称、广告数量、投放起始时间、落地页 URL
→ 将落地页 URL 对应的商品加入候选清单
```

**路径 E — SimilarWeb 相似网站**（可信度 ★★★★☆）【v3 新增】
```
→ 从 Phase 1.3 获取的 SimilarWeb 相似网站列表
→ 访问这些相似网站，查找是否有同品类竞品
→ 这些是"受众重叠度最高"的竞品——同一批用户也逛这些网站
```

**路径 F — 关键词反查**（可信度 ★★☆☆☆，仅补充）
```
→ 分析目标商品标题关键词
→ 搜索同一关键词的其他商品
→ 寻找争夺相同搜索流量的竞争对手
```

### 2.2 竞品分层分类

发现的竞品按以下层级分类，不同层级的分析侧重不同：

```
┌──────────────────────────────────────────────────────┐
│ 直接竞品（同品类 + 相近价格带）                        │
│ → 核心对比对象，分析最深                               │
├──────────────────────────────────────────────────────┤
│ 间接竞品（同品类但不同价格带 / 同价格带但不同子品类）     │
│ → 价格策略和定位参考                                   │
├──────────────────────────────────────────────────────┤
│ 标杆品（品类领导者或高端对标）                          │
│ → 长期学习对象，Listing 最佳实践来源                    │
└──────────────────────────────────────────────────────┘
```

## Phase 3：竞品数据采集

对每个确认的竞品，按以下维度采集数据。根据品类灵活增减维度。

### 3.1 基础信息（页面提取 → ★★★★★）
商品名称/URL、价格/促销、销量/评分、上架时间、店铺信息

### 3.2 Listing 质量（页面分析 → ★★★★★）
标题结构与关键词密度、图片数量/质量/风格、描述逻辑与说服路径、核心卖点、SKU 变体与定价梯度

### 3.3 视觉竞争力评分（★★★★☆）

对每个商品的视觉呈现进行结构化评估：

| 评估维度 | 评分标准（1-5） | 权重 |
|---------|---------------|------|
| 首图吸引力 | 白底=1, 简单场景=2, 精致场景=3, 创意概念=4, 杂志级=5 | 30% |
| 图片数量 | 对比品类最佳实践的充足度 | 15% |
| 信息密度 | 图片中是否嵌入关键卖点信息 | 15% |
| 场景化程度 | 使用场景展示的丰富度 | 20% |
| 视频质量 | 无视频=0, 简单展示=2, 专业制作=4, 品牌级=5 | 20% |

输出一个 1-100 的视觉竞争力综合得分。

### 3.4 营销策略（页面+推理 → ★★★★☆）
定价策略（锚点、心理定价）、促销方式、关键词策略、视觉风格与品牌感

### 3.5 评论区深度采样（★★★★☆）

```
采集协议：
→ 提取最新 15-20 条评论 + 最有帮助的 10 条评论（如平台支持排序）
→ 结构化标注每条关键评论的：评分、核心关键词、情感倾向
→ 统计输出：
  - 好评 Top 3 高频关键词
  - 差评 Top 3 痛点
  - 带图评论的使用场景线索
  - 提及"比XX好/差"的竞品对比线索
  - 复购/推荐意愿线索
```

### 3.6 广告素材查询（★★★★☆）

```
对每个竞品品牌/域名：
→ 访问 Facebook Ad Library：https://www.facebook.com/ads/library/?q=[品牌名或域名]
→ 提取：
  - 当前活跃广告数量
  - 投放起始时间（最早的活跃广告）
  - 素材类型分布（图片/视频/轮播）
  - 文案摘要（前 2-3 条广告的核心文案角度）
  - CTA 类型（Shop Now / Learn More / Sign Up 等）
→ 如发现视频广告，记录视频时长和内容类型（产品展示/UGC/品牌故事）
```

### 3.7 SimilarWeb 竞品受众对比（v3 新增 — 标准/深度模式必做）

```
对每个独立站竞品品牌（非平台卖家）：
→ 访问 similarweb.com/website/[竞品域名]/
→ 提取与Phase 1.3相同的数据字段
→ 与目标品牌进行受众对比：

输出对比表：
| 维度 | 目标品牌 | 竞品A | 竞品B | 品类均值 |
|------|---------|-------|-------|---------|
| 月访问量 | | | | |
| 女性占比 | | | | |
| 最大年龄段 | | | | |
| 主要流量来源 | | | | |
| 跳出率 | | | | |

→ 分析受众重叠度和差异
→ 识别"你的品牌覆盖但竞品没覆盖的受众"和"竞品覆盖但你没覆盖的受众"
```

### 3.8 投放相关推理（推理+估算 → ★★☆☆☆）
推测目标人群、广告可见度、素材风格倾向

- **维度灵活**：不是每个品类都需要所有维度。时尚品需要视觉分析；3C 产品需要规格对比。根据品类调整
- **竞品数量**：3-6 个是最佳范围。少于 3 缺视角，多于 6 稀释重点
- **诚实面对空白**：访问不了的页面、找不到的评论、数据薄弱的环节，明确说出来而不是猜
- **SimilarWeb 降级处理**：如遇到付费墙或访问限制，通过搜索引擎获取缓存数据，或在报告中明确标注缺失
- **用户数据优先**：如果用户提供了订单/广告数据，这些数据的优先级高于一切推断
- **迭代设计**：报告交付后主动询问是否需要深挖某个模块或调整竞品范围

## 已知局限

| 局限 | 影响 | 缓解方式 |
|------|------|---------|
| 无付费工具数据（Jungle Scout 等） | 流量、关键词排名数据缺失 | 用公开数据 + SimilarWeb + Google Trends |
| SimilarWeb 免费版有限制 | 可能被付费墙拦截 | 搜索引擎缓存 + 降级标注 |
| 评论采样有限 | 用户反馈可能不全面 | 优先最新和最有价值的评论 |
| 广告素材只能看 Meta 平台 | Google/TikTok 广告不可见 | Ad Library 覆盖 Facebook + Instagram |
| 单次时间快照 | 无法追踪动态变化 | 建议定期复跑对比趋势 |
| 页面反爬限制 | 部分平台数据提取不完整 | 多种提取方式降级尝试 |
| 人群数据为网站级非商品级 | SimilarWeb显示的是全站受众，不是特定商品受众 | 交叉评论+订单数据校准 |
